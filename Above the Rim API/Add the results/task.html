Description

It's time to implement the game API.

Objectives
  1. Extend the home page in HTML with two paragraphs: /api/v1/games GET all games and /api/v1/games POST add game.
  2. Create a table games in the database:
    1. Column Type

      ID

      Integer

      Primary key

      HOME_TEAM

      Integer

      Foreign Key - teams.id

      VISITING_TEAM

      Integer

      Foreign Key - teams.id

      HOME_TEAM_SCORE

      Integer

      VISTING_TEAM_SCORE

      Integer

  3. Implement the POST method at /api/v1/games that will add a new game to the database after a JSON object is passed. The JSON object should be in the format: 
    {
"home_team": <SHORT>, "visiting_team": <SHORT>, 
"home_team_score": <HOME_TEAM_SCORE>, "visiting_team_score": <VISTING_TEAM_SCORE>
}
  4. If you successfully add a record to the database, the response should be as below with the status code of 201. 
    {"success": true, "data": "Team has been added"}
  5. If the shortcut for the team name doesn't exist in teams, table returns the JSON object as below with the status code of 400. 
    {"success": False, "data": "Wrong team short"}
  6. Implement the GET method at /api/v1/games that collects all the data from the GAMES table and returns it in JSON format with the status code of 200, just like in the example. The key value of the games is ID from the games table.
Examples

Example 1: the response for the home page should look like this.

Welcome to the "Above the Rim" API!

/api/v1/teams GET all teams

/api/v1/teams POST add team

/api/v1/games GET all games

/api/v1/games POST add game

Example 2: response for a POST at /api/v1/games

POST:

{
"home_team": "CHG", "visiting_team": "PRW",
"home_team_score": 123, "visiting_team_score": 89
}

Response:

{
    "data": "Game has been added",
    "success": true
}

Example 3: response for a POST at /api/v1/games if the shortcut name is wrong

POST:

{
"home_team": "XYZ", "visiting_team": "PRW",
"home_team_score": 123, "visiting_team_score": 89
}

Response:

{
    "data": "Wrong team short",
    "success": false
}

Example 4: response for a GET at /api/v1/games

{
"success": true,
"data": {"1": "Chicago Gulls 123:89 Prague Wizards",
         "2": "Prague Wizards 76:67 Chicago Gulls"}
}
