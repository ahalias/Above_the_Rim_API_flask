Description

Let's add the statistics of your favorite teams. In this stage, provide the option to query the API with the single team stats. You will also introduce small check security against adding the team name abbreviation in the wrong format.

Objectives
  1. Extend the home page in HTML with a paragraph: /api/v1/team/%SHORT% GET a team statistics.
  2. Implement the GET method at /api/v1/team/<SHORT> that will return the JSON object with the statistics for the chosen team with the status code of 200 in the format: 
    {
"success": true, 
"data":
       {
        "name": <NAME>,
        "short": <SHORT>,
        "win": <NUMEBR OF WINS AS INTEGER>,
        "lost": <NUMBER OF LOSTS AS INTEGER>}
       }
}
  3. If the team with this SHORT doesn't exist in the database, return JSON with the code of 400. 
    {
"success": false,
"data": "Wrong short format"
}
  4. In the POST method at /api/v1/teams, implement a mechanism that checks whether the short for the added team has a proper format: three capital letters. If it is not, return the JSON object with the status code of 400. 
    {
"success": false,
"data": "Wrong short format"
}
Examples

Example 1: the response for the home page should look like this.

Welcome to the "Above the Rim" API!

/api/v1/teams GET all teams

/api/v1/teams POST add team

/api/v1/games GET all games

/api/v1/games POST add game

/api/v1/team/%SHORT% GET a team statistics

Example 2: response for a GET at /api/v1/team/<SHORT>

{
"success": true,
"data":
       {
        "name": "Prague Wizards",
        "short": "PRW",
        "win": 1,
         "lost": 1
        }
}

Example 3: response for a GET with the team's wrong short name

{
    "success": false,
    "data": "There is no team <SHORT>"
}

Example 4: response for a POST at /api/v1/teams when the shortcut name has the incorrect format

POST:

{
"short": "PRw",
"name": "Prague Wizards"},
}
{
"short": "PRWZ",
"name": "Prague Wizards"},
}

Response:

{
    "success": false,
    "data": "Wrong short format"
}
