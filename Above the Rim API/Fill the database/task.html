Description

The time has come to feed the database with team names and abbreviations. Use the POST method for this. You have decided to provide the GET method with the possibility of obtaining all team names in the JSON format so that everyone can use this data in their systems and websites.

Since you want to continue developing your API in the future, you've decided to number its versions now.

Objectives
  1. Extend the home page in HTML with two <p> paragraphs: /api/v1/teams GET all teams and /api/v1/teams POST add team.
  2. Implement the POST method at /api/v1/teams that will add a new team to the database after a JSON object is passed. The JSON object should be in the format: 
    {"short": <SHORT>, "name": <NAME>}
  3. If you successfully add a record to the database, the response should be as below with the status code of 201. 
    { "success": true,  "data": "Team added"}
  4. There is no need to validate the data;
  5. Implement the GET method /api/v1/teams that collects all the data from the TEAMS table and returns it in JSON format with the status code of 200.
Examples

Example 1: the response for the home page should look like this.

Welcome to the "Above the Rim" API!

/api/v1/teams GET all teams

/api/v1/teams POST add team

Example 2: Response for a POST at /api/v1/teams

POST:

{
"short": "PRW",
"name": "Prague Wizards"},
}

Response:

{
    "data": "Team has been added",
    "success": true
}

Example 3: response for a GET at /api/v1/teams

{
    "success": true,
    "data": {
        "CHG": "Chicago Gulls",
        "PRW": "Prague Wizards",
    }
}
